ประเด็นในที่นี้คือ การย้ายไฟล์ขนาดใหญ่ข้าม OS / เครื่องต่างๆ ผ่าน Thumb drive , Removable Disk , HandyDrive, USBdrive , SD card, external HDD หรือสื่อใดใดก็ตามที่จะใช้

<strong>exFAT คืออะไร</strong>

exFAT เป็น File System ที่เป็นมาตรฐานสำหรับ SD card รุ่นใหม่ๆที่มีความจุตั้งแต่ 32 GB เป็นต้นไป เพื่อแก้ปัญหาเดิมๆที่ FAT เคยมี

<strong>ต่างจาก NTFS อย่างไร</strong>

NTFS เป็นฟอร์แมตที่สร้างขึ้นโดย Microsoft เพื่อแก้ไขปัญหาที่ FAT (ด้วยบิตของ pointer ที่จำกัด ทำให้ไม่สามารถอ้างอิงไฟล์เกิน 4GB ซึ่ง partition รุ่นใหม่ๆสำหรับ Windows ก็คงจะเป็นเช่นนี้อยู่แล้ว (คงไม่มีเครื่องไหนไม่อยากเก็บไฟล์อิมเมจแผ่น DVD สักแผ่นได้)

<strong>*** ปัญหาอยู่ที่ว่า Mac OSX มันจะมอง NTFS เป็น Read-Only สิครับ ***</strong>

แปลว่า สามารถอ่านไฟล์ในนั้นได้ แต่ไม่สามารถเขียน (หรือ ใส่ข้อมูล copy เข้าไป/pasteในนั้น/สร้างfolder ) ได้

(1) ถ้าใช้เป็น external HDD บางยี่ห้อ (เช่น Seagate) จะสามารถลงโปรแกรมเพิ่มเติม ที่แถมมาให้ เพื่อให้เจ้า Mac โยนไฟล์ใส่กันแบบไร้รอยต่อ

(2) OSX จะมีบางโปรแกรมใน app store ที่ "เสียเงิน" แล้วรองรับ ntfs ได้</blockquote>
ถ้าใช้  Journaling for the <em>Mac</em> OS Extended (HFS Plus) <em>file system </em>ก็คงไม่เหมาะสำหรับคนที่จะย้ายไฟล์ข้าม OS เท่าไร

<strong>กลับเข้าสู่ exFAT</strong>

ในเชิงการใช้งาน รองรับการอ่านและเขียน (Read and Write) ใน Windows 8 , Windows 8.1, Mac OSX lion และน่าจะต่อๆไปสำหรับ รุ่นที่ใหม่กว่า

สำหรับ Windows 7 , Vista , XP นั้น โดยปกติ ( Worst case) จะเป็น Read-only (ซะงั้น จริงดิ!!) แต่อย่างน้อยก็ปลอดภัยได้ว่าสามารถอ่านไฟล์ยักษ์ที่บรรจุอยู่ได้

ทาง Linux นั้น มีหลากหลาย distro มากเกินจะหยั่งถึง บ้างก็รองรับทั้งหมด Worst case จะเป็น read-only เช่นกัน (ใน Ubuntu ที่ทดสอบเป็น Read-only) แต่อย่าเพิ่งตกใจ ไม่มีอะไรเป็นไปไม่ได้ในลินุกซ์ :)  วิธีแก้ด้านล่างเลย

<strong>ทำให้เขียนใน Windows 7 ได้ทีเถอะ</strong> (แทนที่ด้วยรุ่นที่ตัวเองใช้)
<blockquote>ทำได้ครับ (โดยไม่ต้อง format ข้อมูลใหม่ด้วย) == ข้อมูลไม่หาย</blockquote>
เพียงเปิด cmd ขึ้นมา (กด start แล้วพิมพ์ cmd / ปุ่ม Windows + R แล้วพิมพ์ cmd / กด start &gt; run &gt; cmd)

จะได้หน้าจอดำๆ  พิมพ์เข้าไปว่า
<blockquote>chkdsk <span style="text-decoration: line-through;">i</span>: /F</blockquote>
สังเกต chkdsk วรรค ตัวย่อไดรฟ์พิมพ์เล็ก: วรรค /F  เป็นเอฟใหญ่

แทน <span style="text-decoration: line-through;">i</span> ด้วย ตัวอักษรย่อพิมพ์เล็กของไดรฟ์ที่เราต้องการ (ที่วินโดว์มองเห็นอยู่ เช่น D: E: F: I: M: ตัวอักษรใดใดที่ปรากฏชื่อไดรฟ์ปลายทางใน My Computer )

เท่านี้ก็จะทำให้ Windows 7/vista/xp รับรู้ถึง exFAT ของเราได้แล้ว ด้วยคำสั่ง check disk เพียงเท่านี้ก็จะเขียนลงไปได้แล้ว

หรือถ้าไม่ได้ต้องการลองแบบ Full options ก็ลอง chkdsk i: /F /R /X ดูได้

&nbsp;

<strong>exFAT กับ Ubuntu</strong>

สั้นๆง่าย ตามนี้เลย สำหรับ Debian/Ubuntu distro
<blockquote>sudo apt-get install exfat-utils exfat-fuse</blockquote>
หลังจากนั้นน่าจะใช้ได้อย่างแน่นอน  (ถ้าไม่ได้อาจจะต้องลอง restart  ดู)
